K6 Performance Testing Guide

A structured overview of K6 Performance Testing, including architecture, execution, and monitoring.

K6 Architecture

K6 is written in Goja, an ECMAScript 5.1 implementation in pure Go. It is highly configurable, script-driven, and integrates with CI/CD pipelines.

  • Does not require Node.js
  • Highly configurable and script-driven
  • Integrates seamlessly with CI/CD
  • Uses goroutines to simulate virtual users
  • Supports distributed load testing using Kubernetes

Running K6

k6 new
k6 run script.js
k6 run --vus 10 --duration 10s simple.js

NYC Taxi Analysis

Domain and Context:

An implementation of my Machine learning's Degree project

Advent of technology in field of transportation especially in the taxi industry have created a several challenges to cab industry. Electronic dispatchment has replaced the traditional VHF radio system. these mobile data terminals are installed in each vehicle typically provides information on GPS localization and distance. This information collected over a period of time will be serving as dataset to analyse and predict the model for future.

Problem:

To improve the efficiency of electronic taxi dispatching systems it is important predict the fare. Given the information on historical rides and fare in centralized system, we need to analyse and predict the near accuracy fare using the optimal model.'

This project involves exploratory data analysis and predictive modeling on NYC taxi datasets using Jupyter Notebooks.

GitHub Repository: NYC Taxi Analysis GitHub Repository

Jupyter Notebook Summaries

1. Data Cleaning and Preprocessing

This notebook focuses on handling missing values, removing outliers, and formatting date-time columns for efficient analysis.

2. Exploratory Data Analysis (EDA)

Includes visualization of ride durations, trip distributions, passenger counts, and geographic mapping of pick-ups/drop-offs.

3. Fare Prediction Model

Develops a machine learning model to predict taxi fares based on trip distance, time, passenger count, and traffic conditions.

4. Time Series Analysis

Performs trend analysis on ride volumes, peak hours, and forecasting future demand using ARIMA and LSTM models.

5. Geographic Heatmap Analysis

Uses folium and geopandas to visualize ride densities and hotspots across NYC.

6. Impact of Weather on Taxi Demand

Analyzes how weather patterns affect ride volume, using external weather APIs for historical data integration.
